The package contains the Network Manager component used within the ZipTie Server. The public API is described by the INetworkManager interface and it's associated dependent objects. The public remote SOAP API is generated from the INetworkManager interface, however the Network Manager is available "in-process" by looking the INetworkManager up in the OSGi service registry.

The following code snippet demonstrates acquiring a reference to the Network Manager through the OSGi framework. We strongly recommend using an OSGi ServiceTracker (not shown here) to manage your service references, rather than acquiring them directly as in this example. 

   private INetworkManager netman;
   ...
   public void start(BundleContext context)
   {
      ServiceReference ref = context.getServiceReference(INetworkManager.class.getName());
      netman = (INetworkManagere) context.getService(ref);
      ...
   }
