The package contains the Scheduler component used within the ZipTie Server. The public API is described by the IScheduler interface and it's associated dependent objects. The public remote SOAP API is generated from the IScheduler interface, however the Scheduler is available "in-process" by looking the Scheduler up in the OSGi service registry.

The following code snippet demonstrates acquiring a reference to the Scheduler through the OSGi framework. We strongly recommend using an OSGi ServiceTracker (not shown here) to manage your service references, rather than acquiring them directly as in this example. 

   private IScheduler scheduler;
   ...
   public void start(BundleContext context)
   {
      ServiceReference ref = context.getServiceReference(IScheduler.class.getName());
      scheduler = (IScheduler) context.getService(ref);
      ...
   }
